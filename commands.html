Git Bash Commands Cheat Sheet

Basic File System Navigation in Git Bash

1. List files and directories:

ls

Options:

  • ls -l – Detailed view (permissions, size, modification date)
  • ls -a – Show hidden files (starting with a dot)

2. Change directory:

cd <path_to_directory>

Examples:

  • cd Documents – Navigate to the "Documents" directory
  • cd .. – Go up one level

3. Go to the user's home directory:

cd ~

4. Go to the previous directory:

cd -

5. Create a new directory:

mkdir <directory_name>

6. Remove an empty directory:

rmdir <directory_name>

7. Remove a file:

rm <file_name>

8. Show the full path of the current directory:

pwd

9. Clear the terminal screen:

clear

10. Move files or directories:

mv <file_name> <destination_path>

Example:

mv file.txt ../ – Move the file one level up.

11. Copy files:

cp <file_name> <destination_path>

Example:

cp file.txt ../ – Copy the file one level up.

Additional Tips:

  • Autocomplete: Use the Tab key to autocomplete file or directory names.
  • Command history: Use the up and down arrow keys to scroll through previously used commands.
